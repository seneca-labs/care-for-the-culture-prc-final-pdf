5. Zip Code Entry

frame-5
Part Number: HIV Prevention_Freddie Experiential Care for the Culture Kiosk_June 2026 US-UNBC-4189

Purpose & Intent

Summary: Collects the user's zip code to determine which county-level HIV and PrEP data to display in subsequent data frames. The zip code is mapped to a county using a static lookup table (no geolocation services). The zip code is not stored after the session ends.

User Action: Enters their 5-digit zip code via the on-screen numpad. Auto-advances after 5 digits are entered. Can skip.

Legal & Compliance Language

Exact copy on screen: "enter your zip code"

Why this language is here: Zip code is the minimum geographic identifier needed to retrieve county-level public health data from AIDSVu. The input is mapped to a county via a static ZIP-to-county lookup and is not stored, transmitted, or combined with other identifiers. No geolocation services are used.

Concerns this addresses:

  • Minimum necessary data collection
  • No persistent storage of geographic identifier
  • No geolocation or GPS used
  • Data used only for session-scoped content personalization

Legal basis: General data minimization principles. HIPAA not triggered (Gilead not a covered entity for this event per Jared's 4/14 assessment).

Copy on Frame (Verbatim)

[user's photo displayed above] enter your zip code [5 digit placeholders: _ _ _ _ _] [numeric keypad] skip

6. PrEP Question

frame-6
Part Number: HIV Prevention_Freddie Experiential Care for the Culture Kiosk_June 2026 US-UNBC-4189

Purpose & Intent

Summary: Asks the user about their relationship to PrEP using de-identified language to shape the narrative experience on subsequent frames. The question uses a two-pathway structure that does not require the user to disclose personal health status.

User Action: Selects one of two pathways or skips the question entirely. Skip preserves user autonomy without penalty.

Legal & Compliance Language

Why this language is here: Two-pathway question structure collapses an earlier four-option design to de-identify users as patients. Phrasing includes "know someone who is" so selecting the option does not imply the user personally is on PrEP. Skip option preserves user autonomy without penalty.

Stakeholder guidance that shaped this:

  • Angelique Bell (Gilead Compliance, 4/14 call): Flagged that the original four-option PrEP question with "I'm on it" created patient self-identification risk under HIV confidentiality statutes and recommended collapsing to ambiguous phrasing that does not identify the user as a patient.
  • Nikki Crump (Majority, 4/14 recap email): Directed simplification to two pathways with flexibility on final wording.

Concerns this addresses:

  • State HIV confidentiality statutes (jurisdiction-dependent)
  • Patient status self-identification risk
  • User autonomy via skip option

Legal basis: State HIV confidentiality statutes (jurisdiction-dependent). HIPAA not triggered per Jared's 4/14 assessment (Gilead not a covered entity for this event). TCPA not applicable to this screen.

Copy on Frame (Verbatim)

this shapes your data card and your morning text does prep show up in your world? i'm on prep or know someone who is not on prep or new to prep skip
